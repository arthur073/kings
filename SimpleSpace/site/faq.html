General

What does SimpleSpace provide?

SimpleSpace is a small subproject of the MASP project, and provides an light-weight, easy-to-use and extensible tuplespace implementation.

What is a tuplespace?

A tuplespace is a bag of tuples, which provides some simple, yet mighty, operations like read, write, remove. A tuplespace is a kind of shared memory, as multiple processes can access it, write into it, etc. Furthermore, tuplespaces provide a template-matching mechanism for their operations. Let's assume there is a tuplespace with entries: (2,"SimpleSpace") and (2, "TupleSpace") . calling read(2, _) would return both entries, as both tuples match the (2, _) template. Calling read(_, "SimpleSpace") would only return (2, "SimpleSpace") .

SimpleSpace's TupleSpaces

What operations does a TupleSpace provide?

A TupleSpace provides following operations:

  • read - reads an element matching a template from space without removing it
  • readAll - reads all elements matching a template from space without removing them
  • remove - removes an element matching a template from space
  • removeAll - removes all elements matching a template from space
  • write - writes an element into space
  • update - updates all elements matching a template
What objects can be put into a TupleSpace?

Basically every Java object can be put into a TupleSpace (also enumerations). However, in order to match the entries with templates (for example in read operations), the object's fields need to be declared public. Non-accessible fields are of course allowed, but will not be matched.

What is the difference between a SimpleSpaces' TupleSpaces and traditional tuplespaces?

The SimpleSpace project defines a TupleSpace interface, which is very similar to a tuplespace. However, there are a few differences:

  • TupleSpace can store any Java object and not only tuples
  • The TupleSpace interface defines additional operations like update, removeAll
  • EventedTupleSpaces (which extend TupleSpaces) provide eventing capabilities
What are all these "wrappers" for?

One of the main problems with existing tuplespace implementations is their size. Many are very heavy-weight. They provide lots of features, but a programmer only uses a fraction of them. While designing SimpleSpace much effort has been put to create a highly configurable and extensible solution. The idea of the wrappers is, that a simple space with only basic features can be wrapped with a proxy space, which adds some functionality. Thus, the programmer can put layers of wrappers upon a space and extend the features according to needs. This mechanism is similar to the Decorator Pattern .

What wrappers exist?

There are several wrappers already developed:

  • Evented Wrapper which provides eventing to any TupleSpace
  • Secure Wrapper which provides some security options
Are SimpleSpace spaces thread-safe?

Yes, all operations can be executed concurrently by multiple threads.
